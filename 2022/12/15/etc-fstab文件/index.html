/etc/fstab文件

发表于 2022-12-15
/etc/fstab文件配置系统初始化时文件系统的挂载点等信息。

作用

用于计录磁盘挂载信息。系统开机时会主动读取该文件，根据读取的配置挂载磁盘。

挂载要求

  1. 根目录必须挂载且必须先于其他挂载点挂载。

  2. 挂载点必须是已经存在的目录。

  3. 挂载点必须遵守系统目录架构原则。

  4. 所有挂载点在同一时间只能被挂载一次。

  5. 所有分区在同一时间只能挂在一次。

  6. 若进行卸载，必须将工作目录退出挂载点（及其子目录）之外。

格式

以行为单位，每行为一条挂载信息，行格式为：

file system

装载了文件系统的磁盘设备文件的路径或Label或UUID

mount point

对应设备的挂载点，其值为目标挂载目录路径。

type

磁盘设备上搭载的文件系统的类型（ext2、ext3、ext4、reiserfs、nfs、vfat、swap等）。

options

挂载使用的参数，其部分值的意义如下：

参数 含义
async/sync 是否以同步方式运行（默认async）
auto/noauto 是否在启动或使用mount -a命令时自动挂载（默认为auto）
rw/ro 以只读/读写模式挂载（默认rw）
exec/noexec 是否允许执行此分区的二进制文件（默认exec）
user/users/nouser 是否允许用户（组）使用mount命令挂载（默认nouser）
suid/nosuid 是否允许SUID操作和设定SGID位（默认suid）
usrquota 启动文件系统支持磁盘配额模式
grpquota 启动文件系统对群组磁盘配额模式的支持
Defaults 对所有参数使用默认值

dump

能否被dump备份命令作用。

0表示不被dump备份命令作用，1为被dump备份命令作用，一般的系统中设为0就行。

pass

开机的过程中，系统默认会以fsck检验文件系统是否完整，该值决定需要检查的文件系统的检查顺序

值 含义
0 不被检查
1 最早检验（一般只有根目录会选择）
2 1级别检验完成之后进行检验
